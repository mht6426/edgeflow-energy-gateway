# 后续规划

> 未实现能力清单与阶段目标。**勿写入简历「已完成」项。**

## 项目硬边界（不做）

- 电力潮流 / 并网保护 / 电网调度算法
- 完整商业 EMS 收益优化
- IEC61850 全量协议栈
- 电池 SOX 算法

## 近期优先（求职补强）

| 项 | 价值 | 复杂度 |
| --- | --- | --- |
| 端到端验证脚本（采集→库→MQTT→补传） | 证明非 Demo | 低 |
| 2h/8h/24h soak 实测记录 | 面试硬证据 | 低 |
| Command Scheduler 超时/重试/回读校验 | 控制闭环深度 | 中 |
| MQTT 用户名密码 | 工控安全基础 | 低 |
| CI ASAN/UBSAN | C 工程质量 | 低 |

## 中期可选

| 项 | 说明 |
| --- | --- |
| MQTT TLS（libmosquitto 可选后端） | 生产环境常见需求 |
| DL/T645 电表协议 | 电力物联网 JD 关键词 |
| 单一扩展策略（TOU / 需量 / 防逆流 三选一） | 不必做全套 EMS |
| 故障注入文档（broker down、串口断开） | 配合 soak 记录 |

## 长期 / 低优先级

- IEC104 完整实现（学习成本高，易做成半成品）
- Thread Pool、多 fd Reactor 拆分 worker
- SIGHUP 配置热加载、HTTP 上报
- `upload_cursor` / `runtime_kv` 表
- 完整状态机（STANDBY/DEGRADED/STOPPED、急停联锁）
