# 开发指南

> 代码规范、部署、硬件选型、测试与长稳记录。

## 1. 代码规范

### 执行环境

| 项 | 约定 |
| --- | --- |
| 开发与运行 | Linux（Ubuntu/Debian 等） |
| 部署目标 | RK3568/RK3588 ARM Linux |
| API | POSIX.1-2008（`_POSIX_C_SOURCE=200809L`） |
| 构建 | 非 Linux 平台 CMake 直接失败 |

### 注释

- 语言：简体中文；术语保留 Mutex、SPSC、POSIX 等英文
- 文件头：模块职责、不负责什么、线程安全说明
- 函数：参数、返回值、调用时机
- 原则：解释「为什么」，少复述代码字面意思

### 模块开发顺序

```text
设计 → 代码 → 单元测试 → 集成验证
```

## 2. 本地构建与测试

```bash
sudo apt install -y cmake build-essential libsqlite3-dev
cmake -S . -B build -DCMAKE_BUILD_TYPE=Release
cmake --build build --parallel
ctest --test-dir build --output-on-failure
```

第三方库 cJSON、libmodbus 已 vendored 于 `third_party/`，无需单独安装。

## 3. 部署

### PC 本地

```bash
./build/edgeflow -c configs/gateway.json
ls -l /tmp/edgeflow
cat /tmp/edgeflow/metrics.prom
```

### ARM64 交叉编译

```bash
sudo apt install gcc-aarch64-linux-gnu cmake
cmake -S . -B build-aarch64 -DCMAKE_TOOLCHAIN_FILE=toolchain/aarch64-linux-gnu.cmake
cmake --build build-aarch64
```

### 板端安装

```bash
scp build-aarch64/edgeflow root@board:/tmp/
scp configs/gateway.prod.json root@board:/tmp/gateway.json
sudo deploy/scripts/install.sh build-aarch64/edgeflow /tmp/gateway.json
sudo systemctl enable --now edgeflow
journalctl -u edgeflow -f
```

### RS485 / libmodbus

`gateway.prod.json` 示例：

```json
{
  "simulate": false,
  "modbus_transport": "rtu",
  "serial_device": "/dev/ttyUSB0",
  "modbus_slave_id": 1
}
```

```bash
sudo usermod -aG dialout edgeflow
ls -l /dev/ttyUSB0
```

Modbus TCP：`"modbus_transport": "tcp"`，配置 `modbus_tcp_host` / `modbus_tcp_port`。

## 4. 硬件选型（摘要）

- **首选**：RK3568/RK3568J 工控板或工控机（双网口、RS485、4GB 内存）
- **不推荐首选 RK3588**：本项目更重工业接口与稳定性，非算力
- **预算方案**：RK3568 SBC + USB-RS485 + PC 端 pymodbus 模拟从站

## 5. 长稳测试记录模板

> 无真实数据前，简历勿写「通过 24h 测试」。

### 环境

- 开发板 / OS / 配置文件 / MQTT broker / 模拟从站

### 方法

```bash
sudo systemctl restart edgeflow
watch -n 5 'cat /run/edgeflow/metrics.prom'
pidstat -p $(pidof edgeflow) 5
```

### 目标指标

| 指标 | 目标 |
| --- | --- |
| 运行时长 | ≥ 24h（可先 2h/8h 渐进） |
| 采集成功率 | ≥ 99.9% |
| RSS | < 64MB |
| systemd 异常重启 | 0 |
| 断网恢复补传 | broker 恢复后 pending 下降 |
